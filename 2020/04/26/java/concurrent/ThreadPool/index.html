线程池

默认的线程池的核心线程不消亡

线程池参数：

  • int corePoolSize ：核心线程池数量
  • int maximumPoolSize：最大线程池数量
  • long keepAliveTime：空闲的线程存活时间
  • TimeUnit unit：时间单位
  • BlockingQueue workQueue：任务队列
  • ThreadFactory threadFactory：线程工厂，设置创建线程的工厂
  • RejectedExecutionHandler handler)：拒绝策略

Executor框架

executor结构主要包括任务、任务的执行和异步结果的计算。

  • 任务：包括被执行任务需要实现的接口：Runnable接口或Callable接口

  • 任务的执行：包括任务执行机制的核心接口Executor，以及继承自Executor的ExecutorService接口。L:Executor框架有两个关键类实现了ExecutorService接口（ThreadPoolExecutor和ScheduledThreadPoolExecutor）

  • 异步计算的结果：包括接口Future和实现Future接口的FutureTask类

  • Executor：一个接口，是Executor框架的基础，把任务的提交于执行分离出来

  • ThreadPoolExecutor：线程池的实现类，执行被提交的任务

  • ScheduledThreadPoolExecutor：一个实现类，给定的延迟后执行命令，或者定期执行

  • Future接口和实现Future接口的FutureTask类：异步计算的结果

  • Runnable和Callable接口的实现类：可以被线程池的实现类执行

Executor的流程：

  1. 主线程首先要创建实现Runnable或者Callable接口的任务对象。

  2. 任务的执行：

    1. Runnable

      1. execute ：ExecutorService.execute(Runnable command)
      2. submit：ExecutorService.submit(Runnable task), Callable task

    2. Callable:

      1. submit：ExecutorService.submit( Callable task)

    3. submit:ExecutorService将返回一个实现Future接口的对象（FutureTask）。

    由于FutureTask实现了Runnable，我们也可以创建FutureTask，然后直接交给ExecutorService执行。

  3. 最后，主线程可以执行FutureTask.get()方法来等待任务执行完成。主线程也可以执行FutureTask.cancel（boolean mayInterruptIfRunning）来取消此任务的执行。

工具类Executors可以把一个Runnable对象封装为一个Callable对象（Executors.callable（Runnable task）或Executors.callable（Runnable task，Object resule））。

ThreadPoolExecutor

  • FixedThreadPool：核心和最大线程池一致，linked的默认队列最长Integer.MAX_VALUE
  • SingleThreadExecutor：核心和最大都为1，linked默认队列最长Integer.MAX_VALUE
  • CachedThreadPool：核心为0，最大Integer.MAX_VALUE，阻塞队列为SynchronousQueue，不存储元素，每一个put必须等待take完，put才结束，期间不能添加新的。

ScheduledThreadPool：

最大Integer.MAX_VALUE，DelayedWorkQueue，延时队列，无界
