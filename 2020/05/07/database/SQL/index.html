SQL

1.1 分类：

  • DDL（ Data Definition Language）：数据定义语言，用来定义数据库对象：库、表、列等；
    • 定义关系模式，删除关系，修改关系模式
    • create/drop/ alter
  • DML（ Data Manipulation Language）：数据操作语言，用来定义数据库记录（数据）；
    • 插入元组，修改元组，删除元组
    • Insert/update/delete
  • DCL（ Data Control Language）：数据挖制语言，用来定义访问权限和安全级别
    • TCL （Transaction Control Languate）
  • DQL（ Data Query language）：数据查询语言，用来查询记录（数据）
    • select

1.2 SQL数据

1.2.1基本类型

  • char（n）：固定长度的字符串

  • varchar（n）：可变长度的字符串，指定最大长度

  • int：整型类型

  • smallint：：小整数类型

  • numeric（p，d）：定点数，有p个数字，d个在小数点右边

  • real，double，precision：浮点数域双精度浮点数

  • float（n）：精度至少为n的浮点数

1.4 SQL查询的基本结构

sql查询的基本结构：select from where

  • select：列出查询结果的包含的属性。
    1. 包含+-*/运算符的表达式，运算的对象可以是常数或元组的属性
  • from：需要访问的关系
  • where：满足特定条件的元组，
    1. 可使用逻辑连接词，and,or,not
    2. 包含比较运算符的表达式,<,<=,>,>=,=和<>

查询的输入在from上指定，在这些关系上进行where与select子句中指定的运算

执行的顺序：from，where，select

from：

  • 输入：关系列表
  • 输出：关系的笛卡尔积

where：

  • from的输入

  • 输出：满足条件的元组

    • 关系运算符
      1. between：a between nums1 and nums2；属性a在nums1到nums2之间的元组
      2. not between：
      3. （a，b） = （c， d）：在属性ab上取值等于c，d的元组
    • 逻辑运算符
      1. and ，or，not：a and b
      2. like
      3. in
      4. is null，is not null

    where a between 10 and 20,(a,b)<(c,d)

select：

  • 输入：where的满足条件的元组
  • 输出：最后的结果，需要观察的属性/表达式 的结果
    1. *：表示所有的属性

显示次序：order by a desc（降序）,b asc（升序）：按照属性a降序，b升序

2.2 事务

事务transaction：由查询和更新居于的序列组成。

一条SQL语句被执行的时候，就隐式的开始了一个事务。

  • commit work：提交当前事务，将该事务所做的更新在数据库中持久保存
  • rollback work：回滚当前事务，撤销该事务所有SQL语句对数据库的更新

事务具有原子性的抽象。

  • 隐式事务：没有明显的开启和结束标记
    • 比如DML语句的 Insert、 update、de1ete语句本身就是一条事务
  • 显式事务：具有明显的开启和结束标记
    • 一般由多条sq1语句组成，必须具有明显的开启和结束标记
    • 步骤取消隐式事务自动开启的功能
    • 1、开启事务
    • 2、编写事务需要的sq1语句（1条或多条）
    • 3、结束事务

2.2.1 4个特性 ACID

  • 原子性atomicity：事务的所有操作要么全部正确执行，要么全部不执行
  • 一致性consistency：隔离事务（无并发事务）时保持数据库的一致性
  • 隔离性isolation：确保事务正确执行，不被并发的事务干扰
  • 持久性durability：一个事务成功完成后，对数据库的改变是永久的，即使出现系统故障

2.3完整性约束：

  • primary key($A_{j1},A_{j2}…A_{jm}$)：声明属性$A_{j1},A_{j2}…A_{jm}$构成关系的主码。主码属性非空且唯一
  • foreign key($A_{k1},A_{k2}…A_{km}$) references tablename：属性参照 tablename的外码
  • not null：属性上的约束，表明该属性不为空。

2.3.1 单个关系的约束

  • auto_increment ：设置属性默认自增长

    • 自增长列要求必须设置在一个键上，比如主键或唯一键
    • 自增长列要求数据类型为数值型
    • 一个表至多有一个自增长列

  • default：设置属性的默认值

  • not null：

    • name varchar（20） not null

  • unique：

    • unique( A,B,C…)：声明指出，ABC。。。形成了一个候选码，候选码属性可以为null（除非声明 not null）

  • check( <谓词> )：指定一个谓词，关系的每个元组都必须满足该谓词。

    • 谓词：包括子查询在内的任何谓词

2.3.2 参照完整性

  • 级联删除
  • 级联更新

foreign key (属性nameA) references tablenameB;

A外码的取值集合必须为B主码取值的子集。
