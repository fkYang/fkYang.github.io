函数依赖：$\alpha \to \beta ,$ 若$\alpha 确定，\beta$ 也确定。对实例中所有的元组对$t_1 , t_2 ,若 t_1[a] = t_2[a]，则t_1[\beta] = t_2[\beta]$

完全函数依赖：在一张表中，若 X → Y，且对于 X 的任何一个真子集（假如属性组 X 包含超过一个属性的话），X ‘ → Y 不成立，那么我们称 Y 对于 X 完全函数依赖。

部分函数依赖：在一张表中，若 X → Y，存在 X 的一个真子集（假如属性组 X 包含超过一个属性的话），X ‘ → Y 成立，那么我们称 Y 对于 X 部分函数依赖

完全：真子集无法确定

部分：存在真子集可以确定

传递函数依赖：假如 Z 函数依赖于 Y，且 Y 函数依赖于 X （严格来说还有一个X 不包含于Y，且 Y 不函数依赖于Z的前提条件），那么我们就称 Z 传递函数依赖于 X

函数依赖在模式上成立：模式的每个元组都满足此函数依赖关系。

第二范式

2NF在1NF的基础之上，实体的属性完全依赖于主关键字（主码）。2NF包含1，并要求有主键；

减去了部份依赖的情况。

第三范式

3NF在2NF的基础之上，属性不依赖于其它非主属性。3NF包含2，且消除冗余数据；

减去了传递函数依赖的情况

具有函数依赖集F的关系模式R属于第三范式的条件是：对$F^+$中所有形如$\alpha \to \beta$的函数依赖，下面至少一项成立：

  • $\alpha \to \beta$是平凡的函数依赖
  • $\alpha$是模式R的一个超码
  • $\beta -\alpha $的每个属性都A都包含于R的一个（某个）候选码中

Boyce-Codd范式

3NF 的基础上消除主属性对于码的部分与传递函数依赖。不允许主键的一部分被另一部分或其它部分决定。

具有函数依赖集F的关系模式R属于BCNF的条件是：对$F^+$中所有形如$\alpha \to \beta$的函数依赖，下面至少一项成立：

  • $\alpha \to \beta$是平凡的函数依赖
  • $\alpha$是模式R的一个超码

BCNF数据库：每个关系模式都是BCNF模式

1、第一范式（1NF）：一个关系模式R的所有属性都是不可分的基本数据项。

2、第二范式（2NF）：关系模式R属于第一范式，且每个非主属性都完全函数依赖于主键。

3、第三范式（3NF）：关系模式R属于第二范式，且每个非主属性都不传递依赖于主键。

4、 BC范式（BCNF）：关系模式R属于第三范式，且每个属性都不传递/部分依赖于主键
